# 用户流程

本目录包含用户完成各种任务的流程设计。

## 用户流程用途

用户流程用于：
- 描述用户操作步骤
- 规划页面跳转路径
- 识别分支和异常处理
- 优化用户体验

## 文档模板

创建新的用户流程文档时，请使用以下模板：

```markdown
# [流程名称] 用户流程

**版本**: 1.0
**最后更新**: YYYY-MM-DD
**设计者**: 姓名

---

## 流程目标

用户想要达成的目标。

## 前置条件

- 用户状态（如：已登录）
- 系统状态（如：有测试数据）

## 流程步骤

1. **[页面名称]** 用户操作 → 系统响应
2. **[页面名称]** 用户操作 → 系统响应
3. ...

## 分支流程

### 成功路径
描述用户顺利完成任务的路径。

### 失败路径
描述可能的错误场景和处理方式。

### 异常处理
- 网络错误
- 权限不足
- 数据无效

## 流程图

\`\`\`
[ASCII流程图或Mermaid图]
\`\`\`

## 相关页面

- 链接到wireframes/中的相关页面

---

**设计者**: 姓名
**评审状态**: 待评审/已评审/已实现
```
