## 设计文档概述

本目录包含测试平台的**产品需求类设计文档**，用于描述产品功能、用户体验和业务流程。

### 适用范围

本目录适用于面向产品和用户体验的设计文档，包括：
- 用户界面线框图（Wireframes）
- 用户流程图（User Flows）
- 高保真原型（Mockups）
- 功能需求示意图
- 业务流程设计

### 与技术设计的区别

| 文档类型 | 存放位置 | 受众 | 内容重点 |
|---------|---------|------|---------|
| **产品设计文档** | `docs/2-requirements/designs/` | 产品、设计、用户 | 需求、体验、流程 |
| **技术UI规范** | `docs/1-specs/ui/` | 开发、技术 | 实现、接口、状态 |

## 设计文档分类

### 1. 线框图 (wireframes/)

**用途**：展示页面布局、元素位置、信息层级

**文档模板**：
```markdown
# [页面名称] 线框图

## 页面功能
简述页面的核心功能和用户目标

## 页面结构
- ASCII艺术线框图
- 或 Figma/Sketch 链接

## 关键元素
1. 元素名称 - 功能说明
2. 元素名称 - 功能说明

## 交互说明
- 点击行为
- 状态变化
- 页面跳转

## 相关流程
- 链接到user-flows/中的相关流程文档
```

**示例参考**：
- 早期设计示意图已归档至 `docs/7-archive/early-designs/ui-wireframes/`
- 可作为参考格式，但需要更新为最新设计

### 2. 用户流程 (user-flows/)

**用途**：描述用户完成特定任务的步骤序列

**文档模板**：
```markdown
# [流程名称] 用户流程

## 流程目标
用户想要达成的目标

## 前置条件
- 用户状态
- 系统状态

## 流程步骤
1. [页面] 用户操作 → 系统响应
2. [页面] 用户操作 → 系统响应
3. ...

## 分支流程
- 成功路径
- 失败路径
- 异常处理

## 流程图
ASCII或Mermaid流程图

## 相关页面
- 链接到wireframes/中的相关页面
```

### 3. 高保真原型 (mockups/)

**用途**：展示视觉设计、品牌元素、详细交互

**包含内容**：
- 带颜色、字体、图标的设计稿
- Figma/Sketch/Adobe XD 链接
- 交互原型演示链接
- 设计资源文件

## 使用指南

### 何时创建设计文档

| 场景 | 文档类型 | 存放位置 |
|-----|---------|----------|
| 需要描述新功能的界面 | 线框图 | `wireframes/` |
| 需要规划用户操作路径 | 用户流程 | `user-flows/` |
| 需要展示最终视觉效果 | 高保真原型 | `mockups/` |
| 需要定义组件技术实现 | 技术规范 | `docs/1-specs/ui/` ⚠️ |

### 设计文档工作流

```
1. 产品需求 (PRD)
   ↓
2. 用户流程设计 (user-flows/)
   ↓
3. 线框图设计 (wireframes/)
   ↓
4. 高保真原型 (mockups/)
   ↓
5. 技术规范 (docs/1-specs/ui/)  ← 技术团队接手
   ↓
6. 前端实现
```

### 文档命名规范

- 使用kebab-case：`test-case-management.md`
- 简洁明确：`login.md` 而非 `login-page-wireframe.md`
- 版本号（如需要）：`dashboard-v2.md`

### 设计评审流程

1. **产品评审**：确认需求覆盖
2. **设计评审**：确认用户体验
3. **技术评审**：确认可实现性
4. **用户测试**：收集真实反馈

## 贡献指南

### 创建新的设计文档

1. **确定文档类型**：线框图/用户流程/高保真原型
2. **创建Markdown文件**：在对应子目录下创建
3. **使用模板**：参考上文的文档模板
4. **添加资源**：如有设计稿，上传到适当位置或链接到设计工具
5. **评审和更新**：定期评审，保持文档与产品同步

### 设计文件存储

- **文本描述**：使用Markdown格式存储在本仓库
- **设计稿**：推荐使用Figma等在线工具，在文档中添加链接
- **图片资源**：如需存储图片，放在 `docs/assets/designs/` 目录
