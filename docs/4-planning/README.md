## 如何使用

### 创建新计划

1. **临时性计划** (Sprint计划、短期任务)
   - 放在 `active/` 目录
   - 完成后移至 `completed/当前季度/`

2. **长期特性计划** (需要多个Sprint完成)
   - 在 `by-feature/` 下创建目录
   - 包含总体计划和分阶段进度

3. **未来规划**
   - 放在 `backlog/` 目录
   - 定期评审并提升优先级

### 归档规则

**每季度末执行**:
```bash
# 1. 将active/中已完成的计划移至completed/YYYY-QX/
mv active/completed-plan.md completed/2024-Q4/

# 2. 更新backlog/roadmap.md

# 3. 清理by-feature/中已完成的子计划
```

## 计划状态定义

| 状态 | 说明 | 下一步 |
|------|------|--------|
| 🟡 计划中 | 计划已制定，等待开始 | 分配资源，开始执行 |
| 🟢 进行中 | 正在执行 | 按计划推进 |
| 🔴 受阻 | 遇到阻碍无法推进 | 解决阻碍问题 |
| ⏸️ 暂停 | 临时暂停执行 | 恢复执行或取消 |
| ✅ 已完成 | 所有任务完成 | 归档到completed/ |
| ❌ 已取消 | 计划取消 | 归档到completed/，注明原因 |
