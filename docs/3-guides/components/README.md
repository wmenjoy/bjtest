# Component Guides

This directory contains implementation and usage guides for UI components in the test management platform.

## Directory Structure

### simple-list-editor/
Complete documentation for the Simple List Editor component:
- Quick start guide
- API reference
- Implementation details
- Verification and testing

## Purpose

Component guides provide:
- **How-to instructions** for implementing and using components
- **Best practices** for component integration
- **API references** with props and configuration options
- **Testing guides** for component verification

## Target Audience

- Frontend developers implementing features
- QA engineers writing component tests
- Technical leads reviewing component architecture
- New team members learning the component library

## Related Documentation

- **Specs**: See `docs/1-specs/ui/` for component specifications
- **Wiki**: See `docs/5-wiki/components/` for business context

## Contributing

When adding new component guides:
1. Create a subdirectory for the component (use kebab-case)
2. Include at minimum: README.md and overview.md
3. Add entry to this README.md
4. Follow the documentation standards in `docs/directory-standards.md`
